Промпт 2:

"Когда нужно указывать generic в useState<User[]>()?"

Результат:
ИИ объяснил, что если начальное значение — пустой массив, TypeScript не может вывести тип автоматически, поэтому необходимо явно указывать generic. Это помогло правильно реализовать:

useState<User[]>(INITIAL_DATA);

Результат:
Получил объяснение использования:

React.ChangeEvent<HTMLInputElement>


и почему нельзя использовать any. Это позволило корректно реализовать handleSearch.

4. Что я узнал (Key Learnings)
Static Typing

Понял разницу между compile-time и runtime проверкой (TypeScript vs PropTypes).

Interfaces vs Type Aliases

interface — для описания формы объекта.

type — для union типов (например, SkillLevel).

Generics

Освоил использование generics в useState<User[]>() и понял, когда TypeScript может вывести тип автоматически, а когда нет.

Typed Event Handling

Научился правильно использовать:

React.ChangeEvent<HTMLInputElement>

React.MouseEvent<HTMLButtonElement>

Теперь понимаю, почему any разрушает безопасность типов.

Conditional Rendering

Закрепил практику безопасного рендеринга массивов и проверки на пустоту.

Project Structure
